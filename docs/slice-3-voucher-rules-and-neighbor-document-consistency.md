# Slice 3 — Voucher Rules + Neighbor-Facing Document Consistency

## Objective

Make voucher redemption rules consistent, prominent, and reusable across request, approval, and neighbor-facing document surfaces.

## Locked Rules

- Voucher expires 30 days after issuance.
- Voucher must be redeemed in one visit.
- Remaining items cannot be saved for a later visit.

## Scope

Included:
- universal request-form rule visibility
- existing receipt/document rule prominence
- future neighbor-facing voucher contract

Excluded:
- multilingual implementation
- delivery links
- PDF/email/SMS
- pricing logic
- redemption logic

## Acceptance Criteria

- rule appears for clothing and furniture vouchers
- approval modal still shows rule
- existing furniture receipt includes prominent rule language
- no pricing logic changes
- no delivery system changes
