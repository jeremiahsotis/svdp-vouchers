# Brainstorming Process

## Rules
- Brainstorming is exploratory, not authoritative.
- Keep language conversational and understandable by non-technical participants.
- Avoid jargon unless explicitly requested.

## Non-Technical Participation Rule
Planning conversations must be understandable by non-technical participants.

Requirements:
- use conversational language
- prefer plain explanations over jargon
- explain technical concepts in ordinary language
- make tradeoffs understandable without code knowledge
- architecture should be expressible in both human and technical terms
