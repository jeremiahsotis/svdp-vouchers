# Slice 3 — Checkpoint 3.1: Universal Request Form Rules

## Objective

Make the 30-day expiration and one-visit redemption rule visible on the Voucher Request Form for all voucher types, not only furniture.

## Required Wording

This voucher expires 30 days after issuance. It must be redeemed in one visit; remaining items cannot be saved for a later visit.

## Scope

Included:
- move or duplicate existing rule so it is always visible
- preserve approval modal rule
- no behavior changes

Excluded:
- receipt updates
- neighbor voucher implementation
- multilingual implementation
- pricing logic

## Acceptance Criteria

- rule visible when Clothing is selected
- rule visible when Furniture is selected
- approval modal rule remains visible
- no submission behavior changed
